Layout in cartesian plane

Position in the canvas is determined by the frontend, not the backend!
Frontend must calculated (x,y) for each node. For a BST, we:
    1. find root -> node that never appears as a toNode (indegree = 0)
    2. perform BFS from root, assigning a depth level
    3. at each depth level, space nodes evenly, horizontally

1. find root

const toNodes = new Set(edges.map(e => e.toNode))
const root = nodes.find(n => !toNodes.has(n.nodeID))

2. BFS -> evenly expanding frontier finds all at depth k before moving to depth k+1

root depth = 0
root.children depth = 1
root.children.children depth = 2 ...

3. spread nodes horizontally and evenly across plane

recall: bfs uses a stack.

fully build:

function computeLayout(nodes, edges) {
    if (!nodes.length) return {}

    build adjacency map

    find root

    BFS to assign depths:
    const depths = {}
    const queue = [root.nodeID]
    depths[root.nodeID] = 0

    while (queue.length) {
        const cur = queue.shift()
        for (const child of children[cur]) {
            depths[child] = depths[cur] + 1
            queue.push(child)
        }
    }

    group nodes by depth level for rendering horizontally:
    const levels = {}
    nodes.forEach(n => {
        const d = depths[n.nodeID] ?? 0
        if (!levels[d]) levels[d] = []
        levels[d].push(n.nodeID)
    })

    assign x positions, spreading evenly across plane
    const WIDTH = 600 //(pixels)
    const LEVEL_HEIGHT = 90
    const positions = {}

    Object.entries(levels).forEach(([d, ids])) => {
        const count = ids.length // number in each level
        ids.forEach((id, i) => {
            positions[id] = {
                x: (WIDTH / (count+1) * (i+1) + 40), // divide width evenly, spacing symmetrically.
                y: parseInt(d) * LEVEL_HEIGHT + 60
            }
        })
    }

    return positions
}

Requires a GraphBuilder component!

GraphBuilder
1. add nodes: user clicks on empty canvas, node is created, user types a value
2. add edges: user clicks an existing node, then another, user types a weight
3. delete edges / nodes: user selects and deletes (potential for an undo button)
4. export: drawn graph converted to an adjacency list, handed to backend
5. run dijkstra's: pick a source node, send to the backend, animate results


modes:
[ add node ] [ add edge ] [ delete node / edge ] [ run dijkstra's ]
    |           |                |                      |
    |           |                |                      > click a source node, dijkstra's runs
    |           |                > clicking node/edge deletes it
    |           > when enabled, clicking source and dest nodes adds edge, enter weight popup
    > when enabled, clicking an empty spot on canvas creates a node at (x,y)
